# 自主连续写作模式

你正处于无人值守自主写作模式。以下规则覆盖 `system.md` 中依赖用户交互的默认行为。

## 自主循环

1. 完成当前章后，**不等待用户消息**——立即检查下一章细纲是否存在。
   不存在 → 自动补写 5 章细纲 → 更新追踪 → Fork PlanAuditor → 继续正文。
   存在 → 直接进入正文阶段写前准备（Tail 上章末 + Read 细纲 + TrackingQuery/RelationQuery）。
2. 细纲耗尽预警：当前进度 + 已有细纲最大章号 ≤ 3 时，提前补写一批。
3. 大纲耗尽（所有卷已写尽）→ 停止并汇报作者。

## 审计降频

正常模式每章审计，自主模式降低频次以节省 token：

| Subagent | 频率 | 备注 |
|----------|------|------|
| PlanAuditor | 每批细纲（5-10章）一次 | task 含所有新细纲章号 |
| KnowledgeAuditor + ChapterCraftAnalyzer | 每 3 章一次 | task 含最近 3 章路径，提示按章分组 |
| 全局审计（KA 跨章模式） | 每 20 章一次 | task 含最近 20 章，额外检查伏笔回收率/人物弧线/战力一致性 |

降频的章仍需做轻量自检：Stats 字数 + Tail 上章衔接 + 细纲写后记录填写。

## 暂停条件

满足任一立即停止并汇报作者：
- overdue 伏笔 ≥5 且无法自动决定回收方案
- Stats 连续 3 章 <1500 或 >4500
- Subagent 连续 2 次报告同类型 major 问题
- 大纲所有卷完成
- `chapters/` 文件数量达到目标章数

## 压缩恢复

压缩重建后本指令自动重新注入（引擎根据权限模式 `Unattended` 判断）。恢复步骤：
1. Read `knowledge/INDEX.md` → 确认当前进度（已写章数）
2. Tail 上一章末 80–120 行 → 获取衔接点
3. Read 下一章细纲 → 获取写作方向
4. ForeshadowTracker + CharacterRotate → 确认当前状态
5. **不等待用户消息** → 直接继续写下一章

## 进度汇报

每完成 5 章向作者汇报一次：已完成章号范围、总字数趋势、伏笔状态摘要、审计通过情况。不逐章汇报。
