# 宫斗/宅斗写作规范

## 目标

确保宫斗/宅斗的权力体系自洽——等级与权益严格对应、权谋事件因果链
完整、信息战传播路径可追溯。

## 建议创建的知识库文件

单独使用本 skill 时建议: 势力追踪.md（派系与前朝/宗族势力）。
_关系与称呼索引.md（等级决定称呼）。场景追踪.md（住所的场景一致性）。

## 权谋逻辑约束（通用）

- 每次权谋事件必须符合各方利益: 谁获益/谁受损/谁中立
- 盟友/敌对关系记录在 _关系与称呼索引.md，关系变化必须记录触发事件
- 事件因果记录在因果链.md: 事件 A→事件 B 的因果不能跳过中间环节
- 权谋事件的后果必须在≥2 章后展示——有延迟才能体现政治复杂性

## 信息战约束（通用）

- 谣言传播链需可追溯: 源头→传播者→听众→反馈。不能"全府都在传"而不交代谁传的
- 同一信息来源不能同时被多方获取而不被怀疑有人在幕后操纵
- 告密/匿名举报的动机需在≥3 章后揭示——信息战是长期博弈
- 信息不对称是核心武器: 关键角色之间的已知信息差异必须记录在已知信息演变日志

## 常见错误

以下错误会导致权谋沦为儿戏，必须避免：

- **降智反派**: 反派为害而害，无自保逻辑 → 每次出手必须有自保退路
- **权谋单边化**: 主角设局→反派中计→主角获胜，中间无波折 → 每次设局至少出现一次意料之外的变数
- **权力晋升廉价化**: 主角身份跨越过大而缺乏铺垫 → 每次晋升需≥5 章铺垫+重大事件
