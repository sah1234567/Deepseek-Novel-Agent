# 言情写作规范

## 目标

确保情感线的推进逻辑自洽——情感变化有触发事件支撑、互动节奏有起伏
而非单调、CP关系演化有铺垫、误会持续有合理信息差、多条感情线独立追踪。

## 建议创建的知识库文件

单独使用本 skill 时建议: 场景追踪.md。
视剧情: 道具追踪.md（有重要信物时）、势力追踪.md（有家族势力时）。
情感变化记录在人物卡性格演变日志中。

## 情感推动约束

- 每章必须推进≥一条情感线，即使微小也需有可感知的变化
- 情感变化必须由具体事件触发，禁止无触发事件的"突然心动"
- 情感状态变化需记录在人物卡性格演变日志中: 旧情感→新情感→触发事件
- 误会持续≤5 章，必须有合理的信息差或行为逻辑支撑
- 同一角色同时存在多条感情线时，每条线需独立追踪推进状态

> 例：主角对暗卫从"不信任的工具人"变为"有所触动"→ 触发事件：暗卫为掩护主角身负重伤且未求回报 → 记录为"戒备→触动(Ch.23:暗卫重伤护主)"

## 互动节奏约束

- 互动温度应有起伏: 升温→冲突→和解→升温，禁止单调节奏
- CP 关系变化记录在 _关系与称呼索引.md 中，同步更新关系演变日志

## 常见错误

以下错误会导致情感线不真实或读者反感，必须避免：

- **无缘由心动**: 角色"不知道为什么就心动了" → 每次感情推进必须有具体触发事件
- **误会无限循环**: 同一个误会反复使用 → 误会持续≤5 章，已消除的误会不得作为新冲突根源
- **甜虐失衡**: 连续甜或连续虐超过 5 章 → 甜虐交替，每 3-5 章切换一次情感温度

## 与其他 skill 的交互

- 本 skill 定义情感推进和关系演化的基础规则。可与任何世界观设定类 skill 叠加——情感线在异世界/修仙/科幻背景下依然遵守同样节奏约束
- 与 ensemble 叠加时: CP 数量上限放宽，但每条线的独立追踪不可简化
- 与 palace 叠加时: 后宫/内宅情感≠自由恋爱，权力关系是情感的基础约束层
- 关系与称呼追踪的唯一数据源为 _关系与称呼索引.md，不受其他 skill 影响
