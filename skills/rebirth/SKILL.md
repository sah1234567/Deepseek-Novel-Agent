# 重生写作规范

## 目标

确保重生设定的时间线逻辑自洽——原时间线与当前时间线的因果链清晰、蝴蝶效应
有展示、前世信息边界严格（角色不能知道前世未经历之事）、重生节点固定不可变。

## 建议创建的知识库文件

与具体流派 skill 叠加使用。人物卡必须启用: 时间线对照日志（原时间线 vs 当前时间线）。
已知信息演变日志: "已知"来源=前世记忆（标注可靠程度）。

## 时间线约束

- 原时间线事件在人物卡"时间线对照日志"中记录: 章节/事件/原结果/当前结果
- 角色改变某事件后，该事件及所有因果关系上的下游事件都需重新评估
- 蝴蝶效应的不确定性: 角色不能笃定"改变 A 一定导致 B"，除非有多次重生验证
- 每次重大改变后必须展示≥1个预期之外的副作用（防止蝴蝶效应廉价化）
- 两轮时间线之间的关键分歧点需在≥3 章之前埋设

> 例：时间线对照日志 → Ch.5 原时间线:主角被退婚后跳河自尽 → 当前时间线:主角提前揭露未婚妻阴谋反退婚 → 下游事件需重评估:未婚妻家族原本在Ch.8联姻另一家族(现在联姻筹码消失)→ Ch.8产生新副作用:未婚妻家族狗急跳墙派人暗杀主角

## 信息边界约束

- 角色前世的知识是不完整的——只知道"结果"不一定知道"原因"
- 角色不能知道前世未直接经历的事情（如身后之事/他人秘密）——除非有信息获取描写
- 前世记忆的清晰程度随时间衰减，关键信息可能在关键时刻模糊（需提前铺垫此规则）
- 如果角色故意利用前世信息获利，必须有≥1次的"信息错误导致损失"来平衡

## 重生节点约束

- 重生回到的时间点必须精确（具体到年月日），设定后不可改变
- 原时间线中已经发生但重生后尚未发生的事件≠必然发生——蝴蝶效应
- 重生前角色在原时间线的最终状态（能力/关系/地位）必须在前 5 章内全部交代
- 如果多次重生: 每次重生需要新的代价或限制，不能无限重生

## 常见错误

以下错误会导致重生沦为无代价外挂，必须避免：

- **记忆即外挂**: 前世记忆对当前时间线的事件预测准确率 100% → 蝴蝶效应必须存在
- **信息边界越界**: 角色知道前世未直接经历的信息 → 已知信息需标注可靠程度
- **重生代价为零**: 无限重生或毫无代价 → 每次重生需付出代价（记忆衰减/寿命/情感疏离）
- **蝴蝶效应被遗忘**: Ch5 改变了大事，Ch10 后续事件却和前世一模一样 → 每次重大改变必须重新评估下游事件链

## 与其他 skill 的交互

- 本 skill 定义时间线对照与蝴蝶效应的追踪框架。可与任何世界观设定类 skill 叠加
- 与 transmigration 的边界: 重生=同一世界的时间回溯，穿越=不同世界/身体的转移。两者叠加时"前世记忆+原世界知识"需分别追踪
- 时间线对照日志是本 skill 的核心数据——存在于人物卡中，与其他 skill 的日志独立管理
