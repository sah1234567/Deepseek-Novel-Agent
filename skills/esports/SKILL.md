# 电竞写作规范

## 目标

确保电竞体系的真实感和逻辑自洽——选手数据变化有训练支撑、战队管理
符合行业规则、比赛描写合理可信，不出现实力无因跳跃或对手降智。

## 开篇约束：必须使用具体游戏

**第一章必须明确主角所玩的游戏名称和核心机制。** 游戏可以是现实存在的（LOL/CS/王者荣耀/星际争霸等）或基于现实改编的（如以 DNF 为原型的"荣耀"），但必须具体到可识别其游戏类型和核心规则。**不能**全程使用"某电竞游戏""某 MOBA"等模糊指代。

## 建议创建的知识库文件

与具体流派 skill 叠加使用。人物卡必开：
- 数值属性演变日志 — 选手核心数据（不同游戏类型的数据项不同，见各 reference）
- 出场记录日志复用为比赛记录 — 每场（对手/赛制/关键节点/结果/MVP）
战队=势力追踪.md（成员/位置/教练组/转会）
可选: 科技树.md — 游戏版本迭代=技术变更

## 选手数据约束（所有游戏类型通用）

- 数值连续性: 核心数据不能跳跃式变化——需≥3 章训练描写支撑
- 每场比赛后数据更新: 旧数据→新数据→变化原因→对应训练内容
- 反应速度/操作精度等生理性数据必须随着年龄/伤病/训练强度合理波动
- 不同游戏类型衡量选手水平的核心数据不同——参考对应 reference 定义

## 战队管理约束（所有游戏类型通用）

- 战队在势力追踪.md 中记录完整阵容: 成员/位置/教练/替补/分析师
- 转会/退役/新人加入记录在势力日志: 触发原因/转会费/对战队战术的影响
- 战术风格一旦确立不可突变——变化需≥5 章铺垫
- 训练体系: 个人训练(操作/意识)+团队训练(配合/战术)+赛前针对性训练

## 比赛描写约束（所有游戏类型通用）

- 每场正式比赛记录: 对手/赛制/关键节点/结果/MVP
- 比赛胜利必须有战术/操作/配合的描写支撑，不能依赖对手失误
- 对手必须塑造成有实力的竞争者，不能突然降智
- 至少 1 次重要比赛失利——失利后需有复盘+调整的描写
- 同一赛季内的实力不能剧烈波动（除非有伤病/转会/内部冲突等触发事件）

## 常见错误

以下错误会导致电竞失去真实感，必须避免：

- **游戏模糊化**: 全程不交代具体游戏名称和规则 → 第一章必须明确具体游戏
- **绝境翻盘常态化**: 每场关键比赛都是落后→翻盘→绝杀 → 至少 30% 的胜利是常规优势
- **对手降智**: 对手在关键时刻做出明显错误决策 → 对手的失误需有合理原因
- **数据悬浮**: 选手数据跳跃式变化 → 数据提升需≥3 章训练支撑
- **版本零适应期**: 游戏版本大改后主角队伍立即适应 → 需≥1 章的适应期

## 与其他 skill 的交互

- 与 sports 共享竞技约束框架，数据类型不同（操作数据 vs 身体数据）
- 与 gaming 的区别: 本 skill=现实中参加电竞比赛；gaming=角色意识进入虚拟游戏世界
- 版本迭代=科技树变更——游戏版本的每次更新记录在科技树.md
