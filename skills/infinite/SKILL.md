# 无限流写作规范

## 目标

确保无限流体系自洽——管理实体的规则声明后不可随意追加、副本难度递进不倒退、
任务结算公平可验、能力积累有明确上限。

## 无限流的核心不变特征

以下特征区分无限流与快穿/多世界穿越：

- **副本制**: 故事以独立副本（任务世界/游戏副本/挑战关卡）为组织单位。每副本有独立规则和退出条件
- **管理实体**: 存在某种控制副本进出和积分结算的力量。无论其形态如何，规则一旦设定不可随意追加
- **生存压力**: 副本有真实的死亡风险。失败扣除积分或降级，积分归零触发不可逆惩罚。副本难度必须递进，不能倒退
- **积累循环**: 进入副本→执行任务→结算积分→返回→消耗积分强化自身→进入下一副本。这个循环是无限流的基本节奏

## 管理中枢的类型

管理中枢是连接各副本的枢纽，首次出场时需明确声明属于哪种类型。根据用户选择的中枢类型，用 Read 加载对应文件：

| 中枢类型 | 特征 | 参考文件 |
|----------|------|----------|
| 共享空间型 | 所有参与者共享的独立物理空间，有兑换/训练/社交设施 | [references/shared-hub.md](references/shared-hub.md) |
| 个人系统型 | 无共享物理空间，通过个人面板完成一切操作 | [references/personal-system.md](references/personal-system.md) |
| 现实锚点型 | 现实世界为基地，副本周期性触发，能力可带回现实 | [references/reality-anchor.md](references/reality-anchor.md) |

三种类型互斥——选择一个，加载对应的参考文件。

## 通用知识库文件

所有类型均需:
- 功法技能.md: 跨副本保留的能力及来源
- 道具追踪.md: 跨副本携带的道具
- 每个参与者独立人物卡，记录属性面板和积分余额

每个副本目录包含:
- INDEX.md — 副本规则（难度等级/主线支线任务/失败惩罚/退出条件/存活人数限制）
- characters/ — 副本内 NPC（通常不可带出副本）
- world-building/ — 副本世界观（若基于已有作品改编，标注与原作差异点）

## 副本世界约束

- 每副本独立 INDEX.md，副本之间规则不可渗透
- 难度必须递进: 后一副本的综合威胁≥前一副本。不能先经历高死亡率副本后又回到新手级
- 每副本退出条件必须在进入时明确公布: 完成主线/存活至时限/击杀指定目标/收集指定物品等
- 副本来源多样: 可取材自已有作品或原创场景。若基于已有作品改编，标注原作与本作差异
- 副本内 NPC 通常不可带出（除非设定允许且消耗大量资源）

> 例：副本"废弃医院"→ INDEX: 难度B/存活率40%/主线:72h内找到隐藏的地下手术室并销毁诅咒源头/支线:①解救被困游魂(+500积分/个) ②找到院长的实验日志(+3000积分+解锁灵异类技能兑换)/失败惩罚:积分-5000/退出条件:主线完成或72h时限耗尽

## 能力积累约束

- 跨副本保留的能力记录在功法技能.md，标注来源副本
- 能力槽/技能栏位需前期设定上限。高等级能力占用更多槽位或消耗更大
- 同类型核心能力不可叠加——核心强化（如血脉/职业/本源力量）只能持有一个，更换需覆盖旧能力
- 属性强化（力量/敏捷/智力等）的每次提升需记录来源（积分兑换/副本奖励/训练成果）

> 例：经过5个副本后，主角能力面板 → 核心强化:中级吸血鬼血脉(副本3获得/覆盖了初级的"骑士训练")/技能槽(上限6): ①基础格斗Lv5 ②血触术Lv3 ③灵视Lv2 ④枪械精通Lv4 ⑤空 ⑥空/属性:力量C+ 敏捷B- 智力C 精神B/积分余额:12000

## 参与者组织（可选）

无限流的参与者组织形式多样，作者需在前期设定中明确：

- **独行模式**: 主角独立行动，副本中遇到的其他参与者可能是临时盟友或竞争对手
- **固定队伍模式**: 参与者自然形成固定队伍，需设定入队/离队规则、积分分配方式
- **组织/势力模式**: 参与者在管理中枢（或现实中）组建大型组织，内部有等级和分工

若使用队伍/组织模式，势力间关系记录在势力追踪.md。死亡参与者的能力不可被直接继承（除非有特定兑换项或遗产机制）。

## 常见错误

以下错误会导致无限流失去核心张力，必须避免：

- **管理实体规则飘移**: 管理实体的规则为应对剧情需要随时追加 → 规则需在 INDEX.md 前期声明，确需追加时需大事件支撑
- **副本难度倒退**: 经历过高死亡率副本后进入低难度副本碾压 → 难度必须递进，不能倒退
- **能力无限叠加**: 每副本获得能力不设上限 → 能力槽前期设定，更换核心能力需覆盖旧能力
- **惩罚虚设**: 积分快要归零时总有紧急任务救场 → 惩罚规则一旦设定严格执行，至少展现一次真实后果

## 与其他 skill 的交互

- 本 skill 与 quick-trans 的本质区别：无限流强调**副本制+难度递进+死亡风险+能力积累上限**，快穿强调**身份切换+任务达成+快速节奏**
- 与 system 叠加时：管理实体≠个人系统。管理实体是公共规则层（中立、不偏袒任何参与者）；个人系统是绑定个体的私有工具（有个性、可辅助宿主）
- 管理中枢为独立世界目录（knowledge/worlds/<中枢名>/），即使中枢是抽象的系统也建议建目录记录其规则
