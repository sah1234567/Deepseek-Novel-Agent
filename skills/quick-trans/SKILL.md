# 快穿写作规范

## 目标

确保快穿多世界切换的节奏和规则一致——每世界五步流程完整、任务失败
有后果、跨世界关系累积可追踪、连续世界后插入缓冲章。

## 建议创建的知识库文件

与具体流派 skill 叠加使用。knowledge/worlds/ 目录下每个世界一个子目录:
- 即使每世界仅停留 1-5 章，也建对应子目录——结构统一
- <世界名>/INDEX.md — 身份/任务/时间/结算规则摘要
- 极短世界（1-2章）的子目录可仅包含 INDEX.md，不建 characters/ 和 world-building/
- 人物卡可选: 数值属性演变日志（任务积分/经验累积）

## 快穿特有约束

- 每世界必须经历: 进入→任务接收→执行→结算→离开，5 步完整
- 世界切换节奏: 每世界结束后的间隙≤1 章缓冲
- 任务失败/部分完成必须有后果: 扣除积分/降级/惩罚世界
- 每世界身份预设记录在 INDEX.md 中

> 例：攻略世界 → 进入(灵魂附身宰相府庶女) → 任务接收(获取太子信任) → 执行(设计偶遇+展露才能+扳倒政敌) → 结算(信任值85/100，部分完成，获2000积分，未触发隐藏任务) → 离开(回归系统空间)

## 节奏管理约束

- 连续≥5 个世界后插入≥1 章的"主空间/休息区"缓冲
- 每世界核心冲突必须在本世界内解决——不能带到下一世界
- 跨世界情感/关系积累需记录在 _关系与称呼索引.md 中

## 主空间管理

- 如果快穿有"主神空间/系统空间"作为中枢: 建 knowledge/worlds/主空间/ 作为独立世界
- 主空间 INDEX.md 声明: 休息规则/兑换规则/世界选择规则

## 任务类型与节奏约束

- 任务类型轮替: 攻略/生存/收集/解密/对抗——同类型任务不能连续出现≥3 次
- 每世界核心冲突的解决方式需变化: 武力/智谋/情感/运气——同一解决方式不能连续使用≥3 次
- 世界难度递进: 后一世界的失败代价≥前一世界。不能先经历高难度世界后又回到新手级
- 长期伏笔: 每 5 个世界中至少有一条跨世界伏笔（某 NPC 跨世界出现/主空间阴谋/身份溯源）

## 常见错误

以下错误会导致快穿沦为换皮流水账，必须避免：

- **世界沦为背景板**: 每世界只换皮（古装→现代→星际），冲突模式不变 → 每世界的核心冲突类型必须不同
- **任务奖励通胀**: 前 3 个世界获得的能力叠加后主角无敌 → 跨世界能力保留需有上限和代价
- **NPC 用完即弃**: 每世界的 NPC 世界结束后不再提及 → 关键 NPC 可在后续世界以不同身份再出现
- **身份切换无痕迹**: 主角在每个世界切换身份后心理无适应期 → 每进入新世界至少 1 段身份适应描写

## 与其他 skill 的交互

- 本 skill 定义世界切换与任务结算的节奏框架。每世界的内部规则由对应世界观 skill 定义
- 与 system 叠加时密切协同——快穿任务系统与系统流规则共用 system 角色卡
- 与 infinite 的边界: 快穿=任务导向+快速切换+跨世界关系累积；无限流=高死亡风险+能力积累+世界难度递进
