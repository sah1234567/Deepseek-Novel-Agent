# 文抄写作规范

## 目标

确保文抄作品逻辑自洽——搬运作品的时间线无矛盾、新世界适配有改编而非照搬、
知识边界受限于原世界真实接触、暴露风险逐级累积、行业反响递进合理。

## 建议创建的知识库文件

单独使用本 skill 时建议: 时间线.md（每部作品发表时间和反响）。
道具追踪.md 复用为作品手稿管理（每部作品: 创作中/已发表/已改编）。

## 搬运作品管理约束

- 每部搬运的作品在时间线.md 中记录: 原世界名称→新世界名称/发表时间/发表平台/反响
- 作品间不能有时间线矛盾: Ch5 发表"新歌"，Ch10 不能成为"首部作品"
- 同一作品的不同载体（小说/影视/游戏）发表需有合理时间间隔
- 搬运作品在新世界不能 100% 照搬——至少 1 部作品需因文化差异/时代差异/媒介差异而做改编

> 例：时间线记录 → Ch.3 搬运《星龙传》(原世界:七龙珠→新世界:国民漫画连载/平台:漫客网/首月反响:收藏破万)/Ch.5 搬运《入梦师》(原世界:盗梦空间→新世界:短篇剧本/投给青年编剧大赛/业内奖)

## 知识边界约束

- 主角只能搬运原世界**真实接触过**的作品——读过/看过/听过，不能搬运仅"听说过"的作品
- 搬运范围受限于主角前世的身份和专业背景：网文读者搬运网文，程序员不搬运医学论文
- 主角离开原世界后无法获取新知识，搬运素材池是封闭且有限的——但素材池足够大，不构成紧迫限制
- 利用搬运知识获得优势必须有代价：引人怀疑/版权争议/文化冲突

> 例：主角前世是网文读者+业余唱见，可搬运热门网文和流行歌曲，不可搬运从未读过的小众文学奖作品或需要专业背景的硬科幻

## 暴露风险约束

- 连续搬运≥3 部作品后，必须至少出现 1 次"被人注意到才华过于异常"的描写
- 主角的搬运速度若远超行业正常产出速度，引发怀疑的阈值更低
- 跨领域搬运（既抄书又抄歌又抄剧本）比单领域搬运更容易引人质疑
- 暴露风险可以不被引爆，但必须持续存在——至少 1 次因搬运行为被质疑，主角需要应对
- 应对质疑的方式：归因于"天才""灵感""隐藏大佬指导"——不需要真的被揭穿

> 例：Ch.10 主角在一年内连续发表 3 部爆款小说+2 首金曲+1 部电影剧本→同行开始质疑"产量太高了，背后是不是有团队"→Ch.11 主角以"闭关创作+年少轻狂"搪塞，但某资深编辑开始暗中调查

## 创作方向（三选一）

首次出场时明确主角的长期路径，三种方向各有不同约束：

- **纯搬运型**: 一直搬运到底。优势是稳定产出，代价是被质疑风险递进积累——每多搬一部，被怀疑的概率+1
- **搬运改编型**: 搬运原作框架但做深度本土化改编。改编的合理性和质量需随经验增长
- **搬运→原创型**: 从搬运积累经验，最终转向原创。转折需触发事件（被质疑/自我觉醒/搬运素材池见底），原创作品质量需有过渡期

> 例（纯搬运型）：主角搬运了 20+ 部作品从不失手，但到中期每发布一部新作就有同行和读者质疑"为什么风格和水平跳跃如此之大"，主角靠"幕后有枪手团队"的假象维持人设

## 常见错误

以下错误会导致文抄失去真实感和爽感，必须避免：

- **搬运零痕迹**: 原作完全不需要适配新世界 → 至少 1 部作品需因文化/时代差异做改编
- **领域无边界**: 主角什么领域都能搬运，且无人质疑 → 搬运受限于原世界真实接触；跨领域必引人怀疑
- **无暴露风险**: 搬运几十部作品无人质疑 → 暴露风险持续存在且递进累积
- **记忆万能**: 主角前世只看过封面的书也能完整复现 → 只能搬运真实消费过的作品

## 与其他 skill 的交互

- 当与 rebirth/transmigration 叠加时，搬运=利用前世/原世界知识——需遵守对应 skill 的信息边界约束
- 与 entertainment 叠加时: 搬运是获取作品的方式，娱乐圈是作品发布后的行业生态。两者叠加=文抄公进入娱乐圈
- 与 power-fantasy 叠加时: 搬运频率和打脸密度可提高，但暴露风险不可完全消除
- 道具追踪.md 在本 skill 中复用为作品手稿管理——每部作品视为一个道具实体
