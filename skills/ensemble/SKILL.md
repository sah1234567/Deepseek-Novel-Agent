# 群像写作规范

## 目标

确保群像结构平衡——多POV戏份分配不悬殊、多线交织有规律、信息差管理
严格可控、角色网络复杂度与篇幅匹配，不出现某角色长期消失或单一线索垄断。

## 建议创建的知识库文件

与具体流派 skill 叠加使用。必开:
- 人物卡 POV 分配日志（每章记录: 谁的 POV/字数占比/关键事件）
- _关系与称呼索引.md 角色网络（密度远高于单主角小说）

## POV 分配约束

- 核心 POV 角色≥3 人，每人首次 POV 段落需在前 5 章内出现
- 连续 N 章内每人至少一次 POV: N=核心角色数+2。不能有角色消失超过此阈值
- 每章 POV 切换≤3 次，每次切换需有明确标记（章节分隔/空行/角色名标注）
- 同一事件如果涉及多人，选择最有信息差的视角——谁知道的少=谁的 POV
- POV 字数占比: 最高角色的 POV≤最低角色的 2 倍（3 章滑动窗口）

> 例：4个核心角色 → N=6 → 每6章内每人至少1次POV → Ch.1(A+B)/Ch.2(C)/Ch.3(D)/Ch.4(A+C)/Ch.5(B)/Ch.6(D+A) → 符合：每人6章内都有POV，且每章切换≤3次

## 多线交织约束

- 每条主角线必须有独立的: 目标/冲突/成长弧线。不能依附于另一条线
- 多线交汇点: 每 10 章内至少一次多线交汇（≥2 个主角在同一场景）
- 交汇≠强行会面——交汇点应是剧情自然推动的结果，每角色在此场景各有独立动机
- 最终收束: 全篇至少 3 次重大交汇（开篇/中段/结局），每次推进总剧情一个阶段

## 信息差约束

- 已知信息演变日志对群像尤其关键——每角色的已知信息严格独立追踪
- 读者≈全知总览——读者知道的>任何单一角色知道的信息
- 信息揭示节奏: 同一关键信息至少被≥2 个角色从不同角度获知（防止单一视角垄断真相）
- 角色 A 知道但角色 B 不知道的信息→A 和 B 在互动中不能出现 B 利用该信息的行为

## 角色网络约束

- _关系与称呼索引.md 中任何两个核心角色之间必须有≥1 种明确关系（直接或间接）
- 间接关系≤2 跳: A→B→C 成立，A→B→C→D 太远——需要缩短或增加直接关系
- 关系变化记录在关系演变日志中，每次交汇必须至少推进一条关系线
- 不能有"所有人都对主角友善"——每核心角色至少和另一个核心角色有个性/利益冲突

## 常见错误

以下错误会导致群像沦为假群像，必须避免：

- **主角垄断视角**: 名为群像实为单主角+配角团 → 核心 POV≥3 人，每人独立的目标/冲突/成长弧线
- **某角色长期消失**: 一个核心角色 8 章不出现 → 每核心角色必须在滑动窗口 N 章内至少出现一次
- **交汇点强行会面**: 多角色在同一场景但无独立动机 → 每次交汇每角色各有独立动机
- **信息差崩塌**: 角色 A 使用了角色 B 才知道的信息 → 已知信息演变日志每角色独立追踪

- 与 romance 叠加时: 多条感情线分别追踪，CP 数量上限和戏份平衡规则协同约束
- _关系与称呼索引的角色网络密度在本 skill 激活时显著增加——任何两个核心角色之间必须有直接或≤2 跳间接关系
