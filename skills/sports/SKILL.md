# 竞技体育写作规范

## 目标

确保竞技体育的真实感和逻辑自洽——选手数据变化有训练支撑、排名/积分
遵守赛制规则、实力波动有伤病/转会/内部冲突等触发事件、对手有战术意图。

## 建议创建的知识库文件

与具体流派 skill 叠加使用。人物卡必开：
- 数值属性演变日志 — 选手数据（不同运动类型的数据项不同，见各 reference）
- 出场记录日志复用为比赛记录 — 每场（对手/赛制/关键节点/结果/MVP）
可选: 科技树.md — 训练方法/器材/营养科学的进步

## 选手数据约束（所有运动类型通用）

- 数值连续性：核心数据不能跳跃式变化——需≥3 章训练描写支撑
- 每场比赛后数据更新：旧数据→新数据→变化原因→对应训练
- 排名/积分的赛季规则需前期设定并全文遵守
- 年龄/伤病/体能对数据的影响需合理推演
- 不同运动类型衡量选手水平的核心数据不同——参考对应 reference 定义

## 运动类型特有约束

不同运动类型的比赛格式、核心数据、管理结构完全不同。根据用户选择的运动类型，用 Read 加载对应文件：

根据用户明确提到的运动类型，只加载对应的参考文件。

## 常见错误

以下错误会导致竞技体育失去真实感，必须避免：

- **主角天赋过载**: 主角同时拥有速度/力量/技术/视野/心理素质全部顶尖 → 必须有≥1 项短板
- **对手降智**: 关键比赛中对手突然失误送分 → 对手的失误需要有合理的战术/心理/体能原因
- **伤病工具化**: 需要困难时安排伤病，不需要时伤病消失 → 伤病必须有恢复周期
- **数据跳跃**: 选手数据跳跃式变化 → 数据提升需≥3 章训练或战术调整支撑

## 与其他 skill 的交互

- 本 skill 与 esports 共享竞技约束框架，数据类型不同（身体数据 vs 操作数据）
- 本 skill 的赛季/排名规则独立于其他 skill 的时间规则——竞技时间线可能与其他线不同步
