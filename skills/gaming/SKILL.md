# 网游写作规范

## 目标

确保网游世界的规则一致性——数值系统连续不跳跃、装备体系归属清晰、
副本配置固定、游戏内外边界不混淆，版本更新有公告铺垫。

## 建议创建的知识库文件

knowledge/worlds/<游戏名>/ — 游戏世界作为独立第二世界:
- INDEX.md — 游戏规则（职业体系/等级上限/死亡惩罚/版本更新机制）
- characters/ — 游戏内 NPC 作为人物卡
- world-building/ — 游戏世界场景/势力/公会
- <副本名>/INDEX.md — 副本配置（怪物/BOSS/掉落机制）
人物卡必开: 数值属性演变日志（HP/MP/EXP/STR/AGI/INT/等级/技能点）
装备=道具追踪.md（属性/耐久/来源/绑定状态）
公会/帮派=势力追踪.md

## 数值系统约束

- 所有数值变化记录在数值属性演变日志中，ConsistencyCheck 验证连续性
- 经验值只能通过游戏内行为获取（杀怪/任务/活动），不能凭空增加
- 等级上限一旦设定不可突破（除非版本更新/资料片，且有提前系统公告描写）
- 属性点/技能点分配后不可重制（游戏内洗点机制需明确设定+代价）

> 例：主角从Lv15升到Lv16 → 日志记录: Lv15(EXP 2400/3000) → 击杀暗影狼×12(+600EXP) → Lv16(EXP 0/3500)，属性点+5(分配至STR+3, AGI+2)

## 游戏世界约束

- 游戏世界在 knowledge/worlds/<游戏名>/ 下独立管理，规则在 INDEX.md 声明
- 游戏内 NPC 作为人物卡: category=game-npc，有自己的性格/目标/记忆
- 副本配置在对应 INDEX.md 中记录，每副本的配置不能在不同攻略次数中随意变化
- 版本更新带来的规则变化需记录并标注更新章号

## 游戏内外边界

- 游戏内货币/物品≠现实货币/物品，严格区分记录
- 游戏死亡=现实死亡 / 游戏伤害反馈现实 / 能力可带回现实 —— 此类边界规则必须在游戏世界 INDEX.md 头部声明，后续不可改变
- 线下交易/代练等行为如果存在，需遵守游戏规则和现实法律约束
- 主角不能在游戏内外拥有矛盾的身份信息而不被发现

## 常见错误

以下错误会导致游戏世界失去真实感和逻辑性，必须避免：

- **游戏内数值悬空**: 主角等级/属性跳跃式增长，无打怪/任务描写 → 所有数值变化记录在数值属性演变日志
- **副本配置漂移**: 同一副本在不同攻略中被描述得配置不同 → 副本配置在 INDEX.md 中固定，只能在版本更新时改变
- **NPC 行为失序**: NPC 在不同章节对同一事件反应不一致 → 游戏 NPC 作为人物卡管理，行为受性格和 AI 规则约束
- **游戏内外边界混淆**: 游戏中的感情/恩怨自动带入现实 → 游戏内外边界规则在首次出场时声明后不可变更

## 与其他 skill 的交互

- 本 skill 将游戏世界作为独立第二世界管理（knowledge/worlds/<游戏名>/）。与多世界管理类 skill 共享 world 目录架构
- 与 esports 的区别: 本 skill=角色意识进入虚拟游戏世界；esports=现实中参加电竞比赛。两者不互斥可共存但不重叠
- 数值属性演变日志与本 skill 绑定——每个进入游戏的角色都需此日志
