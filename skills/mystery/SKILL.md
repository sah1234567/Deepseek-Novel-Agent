# 刑侦推理写作规范

## 目标

确保刑侦推理的逻辑自洽——案件管理无时间线矛盾、线索链完整可回溯、
假线索不误用作推理依据、真凶动机在结局时完整揭露且逻辑自洽。

## 建议创建的知识库文件

单独使用本 skill 时建议: 线索追踪.md（案件/嫌疑人/证据链日志）。
场景追踪.md（案发现场的一致性，多次回访时不能矛盾）。
伏笔追踪.md（每条线索的发现→解读→指向是一条伏笔链）。

## 案件管理约束

- 每起案件在线索追踪.md 中独立记录: 案件ID/状态/受害者/嫌疑人/证据/结论
- 案件状态: 发生→调查→（假破案→）突破→结案。状态变更需有明确触发
- 多案并行时每个案件独立追踪，交叉线索在两案日志中互引
- 案件时间线必须与时间线.md 一致，证据发现顺序不能矛盾
- 连环案件: 每起案件的关联点和差异点需标注

## 线索与推理约束

- 关键线索必须在揭示≥3章之前至少出现过一次（公平推理原则）
- 假线索必须明确标注，防止后续章节误用作推理依据
- 主角推理过程必须可回溯——推理的每个环节都有已揭示线索支撑
- 每案≥3 条线索，其中≥1条为假线索，≥1条指向真相
- 禁止"灵光一闪"式解密——除非此能力提前设定且有代价

> 例：密室杀人案 → Ch.5 主角发现窗台有刮痕(真线索)，现场有一封遗书(假线索-伪造)、地板水渍(真线索指向排水管道)，Ch.8 主角在 Ch.5 的刮痕+水渍基础上推导出凶手从排水管进入，而非依赖 Ch.8 新出现的线索

## 嫌疑人与动机约束

- 每个嫌疑人需在首次引入时提供≥1 个潜在动机（可以是假动机）
- 真凶的动机必须在案件结局时被完整揭露且逻辑自洽
- 不在场证明: 如果使用，必须可验证；如果伪造，必须解释伪造方法
- 嫌疑人数量≤8（超过会导致角色管理困难），核心嫌疑人≤4

## 常见错误

以下错误会破坏推理的公平性和逻辑性，必须避免：

- **线索后补**: 破案时突然出现前文从未提到的证据 → 关键线索必须在揭示≥3 章前出现过
- **假线索误用**: 假线索在后续章节被误当作真线索用于推理 → 假线索必须在日志中明确标注
- **灵光一闪破案**: 主角"突然想到"然后所有线索串起来 → 推理过程必须可回溯
- **凶手人格突变**: 真凶结局动机与之前出场行为完全不符 → 真凶动机必须在前文言行中有伏笔

## 与其他 skill 的交互

- 线索追踪.md 是本 skill 的核心数据文件——线索管理案件内逻辑，伏笔追踪管理跨案件/跨章逻辑。两条统独立但可交叉引用
- 与 supernatural 叠加时: 案件推理需同时考虑常规线索和超自然变量，两者在线索追踪日志中分列
- 当 power-fantasy 激活时: 破案节奏可加快，但公平推理原则不可被覆盖——读者必须有同等机会猜到真相
