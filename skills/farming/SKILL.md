# 种田/基建写作规范

## 目标

确保种田/基建的时序逻辑自洽——建设进度符合现实约束、资源消耗有来源
有去向、科技引进有序递进、人口发展与资源产出匹配。

## 建议创建的知识库文件

单独使用本 skill 时建议: 建设进度.md（城墙/房屋/农田/工厂的物理建造日志）、
科技树.md（农技/水利/工业技术）、道具追踪.md（资源物资）。
人物卡可选: 数值属性演变日志（资源量/产量/人口）。

## 建设时序约束

- 建设必须有时序逻辑: 基础（水源/农田/住房）→生产（作坊/矿场）→升级（工厂/城池）
- 每项建设在建设进度.md 中记录: 开始章/完成章/规模/消耗/产出
- 同时进行的建设项目≤3 个（人力/资源分散注意力的现实约束）
- 建筑升级不能跳过中间阶段: 不能从茅草屋直接升级到砖石结构
- 建设时间需合理: 城墙不能一章建成（除非有魔法/科技/人力支撑）

> 例：建设木栅栏围墙(Ch.5-6, 消耗木材×200+人工×20) → 升级石墙(Ch.15-17, 前提:采石场建成+石灰窑投产, 消耗石料×500+人工×50) → 不能跳过木栅栏直接建石墙

## 资源与科技约束

- 资源消耗必须有来源记录: 从何处获取/数量/消耗去向
- 科技引进（穿越种田特需）记录在科技树.md: 引进章/来源/前置依赖
- 技术不能跳跃时代: 不能从青铜时代直接跳到电气时代（除非有穿越者知识+量产条件）
- 人口增长与资源产出成正比——不能突然出现大量劳动力或消费者

## 规模与影响约束

- 建设规模扩大后必须带来管理复杂度: 领地越大→管理越难
- 建设对外界的影响: 邻国/邻居的反应、资源竞争、技术泄露风险
- 建设成品的维护和折旧: 建筑会老化，水利需疏浚，农田需轮作

## 常见错误

以下错误会导致种田失去真实感，必须避免：

- **建设速度无视现实**: 城墙一章建成、作物一周收割 → 每项建设需记录开始章/完成章/消耗/产出
- **资源凭空循环**: 收入永远大于消耗 → 至少 1 次资源危机
- **技术跳跃时代**: 从青铜器直接跳到蒸汽机 → 科技树.md 记录技术引进的时序和前置依赖
- **人口工具化**: 需要时突然出现大量劳动力 → 人口增长与资源产出成正比

## 与其他 skill 的交互

- 本 skill 定义建设时序和资源管理的约束框架。可与任何世界观设定类 skill 叠加
- 与 apocalypse 叠加时: 建设=重建，资源消耗逻辑加倍严格
- 建设进度.md 与本 skill 绑定——每个建设项目在此文件中追踪生命周期
