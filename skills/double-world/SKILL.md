# 两界穿越写作规范

## 目标

确保两界穿越体系自洽——双世界独立设定不交叉污染、往返规则前后一致、
资源互通逻辑不出现矛盾，世界差异维度始终恒定。

## 建议创建的知识库文件

与具体流派 skill 叠加使用。knowledge/worlds/ 目录下每世界一个子目录:
- <世界名>/INDEX.md — 世界规则摘要（含往返限制/携带规则/时间流速）
- <世界名>/characters/ — 此世界的原住民
- <世界名>/world-building/ — 此世界的背景/势力/时间线

穿越者本人放在 knowledge/characters/（共享层），不放在任何 worlds 子目录下。
大纲.md 每行加"所在世界"列。_关系与称呼索引.md 每行加"所在世界"列。

## 双世界管理约束

- 每世界独立 INDEX.md，在头部声明该世界规则: 穿越触发条件/限制/能否往返/时间流速差异/携带物资限制
- 两世界共享的设定（如贯穿两界的上古传送阵）放在 knowledge/world-building/ 共享层
- 每世界的背景设定/势力/时间线独立——World A 的势力关系不影响 World B
- 两个世界的差异维度（物理/社会/语言/科技等级）必须在各 INDEX.md 中声明

> 例：现代都市 ↔ 修仙界 → 现代 INDEX 声明"科技侧/无灵力/时间1:1/仅灵石可携带往返"；修仙界 INDEX 声明"修真侧/灵力充裕/时间1:1/灵石+附灵物品可携带往返"

## 穿越约束

- 往返限制和代价在各世界 INDEX.md 中明确，设定后不可改变
- 穿越携带物资/知识的限制由各世界规则定义
- 穿越后时间流速差异（如果有）必须恒定——不能中间突变
- World A 的规则不能突然在 World B 生效——通行需要穿越规则明确允许

## 世界差异约束

- 两个世界的差异维度（物理/社会/语言/科技等级）必须在前 3 章内全部展示
- 每世界的世界观不能交叉污染——A 界的地理/势力不能直接作用于 B 界

## 常见错误

以下错误会导致世界观矛盾和读者困惑，必须避免：

- **世界差异扁平化**: 两个世界仅在"有没有魔法"上有区别，社会/文化/价值观完全一样 → 至少 3 个差异维度需在前 3 章全部展示
- **携带规则不一致**: Ch3 可以带灵药回现代，Ch10 同类型物品突然不能带 → 携带规则在 INDEX.md 头部声明后不可变更
- **时间流速突变**: 修仙界 1 天=现代 1 年，中间突然变成 1 天=1 小时 → 时间流速差异一旦设定必须恒定
- **世界规则渗透**: A 世界的功法突然在 B 世界生效而无穿越规则说明 → 跨世界规则通行需在穿越规则中明确允许

## 世界导航

在双世界之间切换工作时，遵循以下步骤：

1. **发现目标世界的结构**
   ```
   Glob knowledge/worlds/{目标世界名}/**
   ```
   确认该世界下有哪些文件（INDEX.md、characters/、world-building/ 等）。

2. **阅读世界规则摘要**
   ```
   Read knowledge/worlds/{目标世界名}/INDEX.md
   ```
   了解该世界的核心设定、势力分布、时间线、穿越规则。

3. **了解该世界已有角色**
   ```
   Glob knowledge/worlds/{目标世界名}/characters/*.md
   ```
   对关键角色 Read 其人物卡。穿越者角色位于共享层 `knowledge/characters/`。

4. **回退规则**
   如果世界目录下缺少某文件（如因果链.md、伏笔追踪.md），回退到共享层 `knowledge/plot/` 的同名文件。

5. **写前检查**
   在写涉及跨世界的章节前，先 Read 两个世界的 INDEX.md 确保规则一致性。
