# 机甲写作规范

## 目标

确保机甲体系完整自洽——机体属性连续、装备升级有来源、战斗损耗有维修
周期、驾驶员与机体的适配度前后一致，不出现性能无因跃升。

## 建议创建的知识库文件

与具体流派 skill 叠加使用。机体=道具追踪.md（属性/装备/状态/归属）。
人物卡可选: 数值属性演变日志（驾驶数据/战绩）。
科技树.md（机甲技术迭代）。

## 机体管理约束

- 每台机甲在道具追踪.md 中独立记录: 型号/尺寸/核心参数/装备槽/状态/驾驶员
- 机体升级换代→旧机去向必须记录（退役/损毁/转让/拆解）
- 驾驶员更换→道具持有者变更
- 同一型号的机甲存在个体差异时需记录差异来源（技师调校/战场改装）

> 例：机甲"苍雷"(型号:MKI-IV/参数:出力8500kW+装甲厚120mm+推进力12G/装备槽:等离子刀+粒子盾/状态:右臂装甲Ch.22战斗受损→Ch.23-24维修中/驾驶员:叶辰)

## 战斗与损耗约束

- 机体性能=上限，驾驶员技术=发挥程度
- 每次战斗损耗记录在道具日志: 损伤部位→维修方式→维修时间→性能影响
- 维修时间需合理: 小修≤1天，大修≥3天，核心损伤可能需要整章修复
- 关键部件损坏后替换件的来源必须追溯
- 能量/燃料消耗需记录: 每次出击的消耗量和补给周期

## 驾驶员与机体的关系约束

- 驾驶员与机体的适配度（精神力/体能/反应速度）需前期设定
- 机体特殊性（如神经链接/意识同步）需设定风险和副作用
- 驾驶员的身体负荷: 高G机动/长时间驾驶的生理代价
- 驾驶员退役/受伤后机体的去向和影响

## 常见错误

以下错误会导致机甲体系失去机械真实感，必须避免：

- **机体性能无上限**: 主角机体无限升级，无性能天花板 → 每代机体的性能上限在道具追踪中声明
- **维修时间压缩**: 关键部件损坏→一章内修复→马上重返战场 → 维修时间需合理
- **驾驶员无负荷**: 长时间高强度驾驶无任何生理代价 → 弹药/能量消耗、G力负荷、神经链接损耗需记录
- **机体回退悬空**: 旧机退役后去向不明，需要时又突然出现 → 每台机体的退役/损毁/转让/拆解需在道具日志中记录

## 与其他 skill 的交互

- 本 skill 的机体作为道具追踪（道具追踪.md），驾驶员作为人物卡管理——机体属性≠角色属性
- 与 scifi 叠加时: 机甲技术迭代记录在科技树.md，每一代升级记录解锁条件和前置依赖
- 驾驶员与机体的适配度是独立维度——不受战力/修炼体系约束
