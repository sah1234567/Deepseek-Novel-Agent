# 大纲

**分卷大纲只写本卷整体剧情**——描述本卷起止状态、主线冲突、核心转折和结局方向。**禁止**逐章展开写具体情节或详细场景。

> **每卷「本卷概要」必须包含以下四个要素，缺一不可：**
> 1. **起点状态**：本卷开始时主角在哪里、什么处境
> 2. **核心冲突**：对手/障碍是什么
> 3. **关键转折**：中点/高潮位置（本卷最重要的转折事件）
> 4. **终点状态+钩子**：本卷结束时主角获得了/失去了什么，为下一卷留下什么钩子
>
> 四要素不完整 → 视为该卷大纲未完成 → **禁止创建该卷任何细纲**。可多稿迭代（`plan/大纲-v*.md`），但定稿必须满足四要素。
>
> **修改本大纲后的级联检查清单（由 revision skill 触发）：**
> 1. 受影响卷的所有已有细纲 → Read 场景拆分，判断是否仍与大纲概要一致；不一致的标记需修改
> 2. 已写章细纲与正文不一致 → 标记需重写的章，AskUserQuestion 确认后处理
> 3. 未写章细纲 → 直接 Edit 对齐新大纲，改完后 Fork PlanAuditor 重新审计
> 4. 伏笔追踪中跨卷伏笔 → ForeshadowTracker 检查是否受影响
> 5. 因果链 → PlotGraph 检查跨卷因果边是否断裂
> 6. 全部修改完成后 → Fork PlanAuditor 审计受影响细纲
